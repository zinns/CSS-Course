Resumen de Git & Github

Git es un controlador de versiones en el cual puedes trabajar en repositorios de manera local o remota, tambien puedes hacer cambios en algún punto de la historia de un proyecto en caso de algún error, git permite empezar a trabajar desde cualquier punto en la historia del proyecto.

Las ramas son importante para llevar un orden en el proyecto con los puntos de la historia y saber desde donde se añade cada cosa y donde se hace otra cosa, Ademas que esta pueden subir a la nube en github para poder compartirse con otros desarolladores de manera remota.

Los puntos en la historia se usan para trabajar desde la version mas vieja del proyecto a la version mas actual del mismo, Para crear estos puntos se usan los commits los cuales ayudan a dar mas informacion sobre lo que se hace en este punto de la historia.

Hay ramas primarias y ramas complementarias , tenemos main o master y develop que son ramas primarias y luego estan las complementarias que se usan para añadir o modificar cosar de un repositorio.
